# Trae（给 AI 的启动上下文）

目标：让 AI 在每次开始工作前，快速获取本仓库的产品目标、模块边界、工作流程与编码规范。

## 1. 项目快照（先读这一段再开始改）

- 产品：家教直聘（BOSS 直聘式）—— 学生/家长发需求，老师刷需求并直接开聊，促成预约/成交
- 技术栈：
  - 前端：Vue 3 + Vite + TypeScript + Pinia（`ai-tutor-web/`）
  - 后端：Maven 多模块 Spring Boot（聚合入口 `ai-tutor-starter/`）
  - 存储：MySQL + Redis（可选 RocketMQ）
- 关键文档：
  - 后端启动与联调：README-backend.md
  - MVP 规划与现状：docs/plan.md

## 2. 模块边界（不要搞错改动位置）

- 前端：`ai-tutor-web/`（页面 `src/pages/`，路由 `src/router/`，store `src/stores/`，API `src/api/`，测试 `src/test/`）
- 预约/业务服务：`tutor-appointment-service/`
- IM 服务：`videoCall-IM-service/`
- 聚合启动：`ai-tutor-starter/`
- 初始化 SQL：`sqlDoc/huoyue.sql`

## 3. 当前默认目标（短记忆：按里程碑推进）

以 docs/plan.md 为唯一口径；默认先把 MVP 做“可用”，按 A→D 顺序推进：

- A 登录分流 + 我的资料：⬜ 未开始 / ⬜ 进行中 / ⬜ 已完成
- B 学生端发/编需求：⬜ 未开始 / ⬜ 进行中 / ⬜ 已完成
- C 教师端需求广场/搜索/详情：⬜ 未开始 / ⬜ 进行中 / ⬜ 已完成
- D 开聊 + 会话列表 + 聊天页：⬜ 未开始 / ⬜ 进行中 / ⬜ 已完成

非目标（默认不做）：支付/订单履约/推荐算法/多媒体消息/复杂风控/实时推送等（避免范围爆炸）。

## 4. 工作方式（OpenSpec 风格，精简版）

开始写代码前，必须先产出这 4 件事并对齐：

- Spec：要做什么 + 不做什么 + 验收标准（可判定）
- Tasks：按模块/文件拆任务，明确改动面
- Checklist：测试/类型/安全/回归路径
- Verify：给出可复现验证方式（命令/页面路径/接口请求）

完成改动后，必须补齐“会话日志”，并在里程碑进度里更新状态。

## 5. 编码与注释约定（够用即可）

- 先遵循现有风格与依赖，不随意引入新库；复用已有工具与模式
- 不泄露敏感信息：不记录/打印 token、验证码、密钥、真实手机号、连接串密码
- 错误处理要一致：不吞异常；对外返回/对内提示要可定位
- 变更必须可验证：最少补 1 条可运行的验证（单测/接口请求/页面回归路径）
- 注释策略：默认不写；仅在“为什么/边界/权衡”需要说明时写，且保持可维护
