## ADDED Requirements

### Requirement: 学生可浏览教师列表并按条件筛选
系统 MUST 提供学生端教师列表页，以列表-详情工作台形式展示教师，并支持关键词与基础筛选条件进行查询。

#### Scenario: 学生进入教师列表页看到默认列表
- **WHEN** 学生已登录并进入教师列表页
- **THEN** 系统 MUST 展示教师列表的第一页数据，并展示“加载更多”能力（当存在更多数据时）

#### Scenario: 学生使用关键词搜索筛选教师
- **WHEN** 学生在教师列表页输入关键词并触发搜索
- **THEN** 系统 MUST 使用关键词刷新教师列表，并使刷新后的列表与关键词筛选一致

#### Scenario: 学生使用筛选条件筛选教师
- **WHEN** 学生在教师列表页选择授课方式/城市等筛选条件并触发刷新
- **THEN** 系统 MUST 使用筛选条件刷新教师列表，并使刷新后的列表与筛选条件一致

#### Scenario: 学生加载更多教师
- **WHEN** 学生在教师列表页点击“加载更多”且存在下一页
- **THEN** 系统 MUST 追加加载下一页教师数据，并保持当前筛选条件不变

### Requirement: 学生可查看教师详情并发起操作
系统 MUST 支持学生在教师列表页选择教师并查看详情，并在详情区域提供收藏与发起申请入口。

#### Scenario: 学生选择教师查看详情
- **WHEN** 学生在教师列表页点击某个教师列表项
- **THEN** 系统 MUST 在详情区域展示该教师的详细信息

#### Scenario: 学生收藏或取消收藏教师
- **WHEN** 学生在教师详情区域点击“收藏/已收藏”
- **THEN** 系统 MUST 更新该教师的收藏状态并在 UI 上同步展示

#### Scenario: 学生从教师详情发起申请
- **WHEN** 学生在教师详情区域点击“发起申请”并确认发送
- **THEN** 系统 MUST 创建或获取与该教师的沟通会话并发送申请内容，且 MUST 跳转到聊天页面

### Requirement: 非学生角色不可使用教师列表页
系统 MUST 限制教师列表页仅对已登录学生可用。

#### Scenario: 未登录访问教师列表页被引导登录
- **WHEN** 未登录用户访问教师列表页
- **THEN** 系统 MUST 引导用户前往学生登录页

#### Scenario: 教师角色访问教师列表页被纠正跳转
- **WHEN** 教师已登录但访问教师列表页
- **THEN** 系统 MUST 纠正跳转到教师端需求页
