## ADDED Requirements

### Requirement: 管理员可分页查询付款记录
系统 MUST 为管理员提供付款记录分页查询能力，支持按时间范围、状态、支付渠道、订单号与业务关联条件进行过滤检索。

#### Scenario: 管理员按时间范围分页查询
- **WHEN** 管理员输入时间范围并请求付款记录列表
- **THEN** 系统 MUST 返回分页结果并按创建时间或支付时间排序

#### Scenario: 管理员按订单号精确查询
- **WHEN** 管理员输入商户订单号（orderNo）进行查询
- **THEN** 系统 MUST 返回对应付款记录或明确提示不存在

### Requirement: 管理员可查看付款记录详情用于对账与排障
系统 MUST 允许管理员查看单笔付款记录的详情字段，包括业务关联、金额、渠道、第三方单号/交易号、状态迁移时间与回调接收信息等用于对账与排障的关键信息。

#### Scenario: 查看付款记录详情
- **WHEN** 管理员从列表进入某笔付款记录详情
- **THEN** 系统 MUST 展示该付款记录的关键字段与回调处理结果（不包含任何密钥类敏感信息）

### Requirement: 管理端接口必须受鉴权保护
管理端付款记录查询与详情接口 MUST 进行管理员鉴权与权限校验，未授权请求不得获取任何付款数据。

#### Scenario: 未授权访问被拒绝
- **WHEN** 未携带合法管理员身份的请求访问付款记录接口
- **THEN** 系统 MUST 拒绝请求并返回未授权或无权限错误
