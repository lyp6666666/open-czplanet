## ADDED Requirements

### Requirement: 支付闭环关键逻辑必须具备单元测试覆盖
系统 MUST 为支付域的关键逻辑提供单元测试，覆盖验签/校验、订单状态机幂等、回调重复投递等主要分支。

#### Scenario: 回调幂等的单元测试用例
- **WHEN** 相同支付订单收到多次相同的支付成功回调
- **THEN** 单元测试 MUST 验证订单仅发生一次从待支付到成功的状态迁移，且事件仅发布一次

### Requirement: 必须提供可重复执行的闭环集成测试
系统 MUST 提供可在本地与 CI 环境重复执行的集成测试，通过 Mock YunGouOS 下单返回与回调通知完成“下单→出码→回调→查单→事件→业务解锁”的闭环验证。

#### Scenario: 集成测试验证从下单到业务解锁
- **WHEN** 集成测试触发统一下单并模拟 YunGouOS 回调成功
- **THEN** 集成测试 MUST 验证支付单状态为成功、支付成功事件被消费、业务订单被更新为已支付

### Requirement: 测试数据与敏感信息必须隔离与脱敏
测试实现 MUST 避免依赖真实支付密钥与真实第三方环境，并对日志与断言中的敏感信息进行脱敏处理。

#### Scenario: 测试不使用真实密钥
- **WHEN** 执行支付相关单元测试与集成测试
- **THEN** 测试 MUST 使用测试配置或 Mock 签名材料，且不需要连接真实 YunGouOS/支付渠道
