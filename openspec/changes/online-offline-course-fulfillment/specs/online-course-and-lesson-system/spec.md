## ADDED Requirements

### Requirement: 线上授课必须基于长期课程承接
系统 MUST 将线上授课建模为长期课程 `Course`，而不是直接把每一次上课当成独立交易。

#### Scenario: 线上创建课程
- **WHEN** 学生或教师在线上沟通通过后发起创建课程
- **THEN** 系统 MUST 创建一个长期课程实体，并记录授课双方、科目、目标、单节价格、时长和固定课表

### Requirement: 课程创建后自动生成试课和未来课节
系统 MUST 在长期课程创建成功后自动生成第一节试课，并根据每周固定时间滚动生成未来课节。

#### Scenario: 创建长期课程成功
- **WHEN** 课程被双方确认并生效
- **THEN** 系统 MUST 自动生成一节 `TRIAL` 类型课节，并生成未来周期的普通课节

### Requirement: 我的课程展示长期课程
系统 MUST 在“我的课程”页展示长期课程列表，而不是直接平铺所有单节课。

#### Scenario: 进入我的课程页
- **WHEN** 学生或教师打开“我的课程”
- **THEN** 系统 MUST 展示长期课程卡片，并显示课程状态、下一节课时间和待办信息

### Requirement: 用户可以在课程详情中管理单节课
系统 MUST 在课程详情页提供查看和管理该课程下所有单节课的能力。

#### Scenario: 查看课程详情
- **WHEN** 用户进入某个长期课程详情页
- **THEN** 系统 MUST 展示该课程的单节课列表，并提供加课、删课、调课等操作入口

#### Scenario: 调整单节课
- **WHEN** 学生或教师对某一节课发起调课
- **THEN** 系统 MUST 仅修改该课节，除非用户明确选择同步修改长期固定课表

### Requirement: 系统支持课程暂停与结束
系统 MUST 支持长期课程的暂停与结束状态，以承接阶段性停课、退课和自然结课。

#### Scenario: 暂停课程
- **WHEN** 学生或教师对长期课程发起暂停
- **THEN** 系统 MUST 暂停后续自动排课，并保留课程历史记录

#### Scenario: 结束课程
- **WHEN** 课程被正常结课或退课终止
- **THEN** 系统 MUST 将课程标记为结束态，且 MUST NOT 再生成新的课节
