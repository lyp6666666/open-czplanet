## ADDED Requirements

### Requirement: 线上课程首节课必须为试课
系统 MUST 将线上课程的第一节课标记为试课，并据此承接继续/退课的后续流程。

#### Scenario: 创建线上课程
- **WHEN** 线上长期课程创建成功
- **THEN** 系统 MUST 自动生成一节 `TRIAL` 类型课节作为第一节课

### Requirement: 试课后学生必须做继续或退课判断
系统 MUST 在试课结束且试课费用支付后，要求学生显式选择“继续课程”或“退课”。

#### Scenario: 试课已完成且已支付
- **WHEN** 学生完成试课支付
- **THEN** 系统 MUST 向学生展示“合适/不合适”的决策入口，且在做出决定前不应默认推进长期课程

### Requirement: 试课合适后课程继续
系统 MUST 在学生确认试课合适后，将长期课程推进到正常进行中状态。

#### Scenario: 学生确认合适
- **WHEN** 学生对试课做出“合适，继续上课”的选择
- **THEN** 系统 MUST 将课程状态更新为 `ACTIVE`，并保留后续课表与聊天权限

### Requirement: 试课不合适后课程终止并关闭聊天
系统 MUST 在学生确认试课不合适后终止课程，并关闭该课程关联聊天的继续沟通能力。

#### Scenario: 学生确认不合适
- **WHEN** 学生对试课做出“不合适，退课”的选择
- **THEN** 系统 MUST 结束课程、取消未开始课节，并将关联聊天关闭或转只读

### Requirement: 课程结束后不得继续排课
系统 MUST 在课程结束或退课后阻止继续约课、调课、加课和进入课堂。

#### Scenario: 退课后尝试继续约课
- **WHEN** 某课程已处于 `DROPPED` 或 `FINISHED` 状态
- **THEN** 系统 MUST 阻止新的课节操作与课堂准入
