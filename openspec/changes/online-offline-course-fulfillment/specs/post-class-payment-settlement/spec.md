## ADDED Requirements

### Requirement: 线上课节必须逐节课后支付
系统 MUST 在每一节线上课完成后生成该课节对应的待支付订单，并要求学生在平台内支付。

#### Scenario: 课节完成
- **WHEN** 一节线上课被标记为已完成
- **THEN** 系统 MUST 生成一笔待支付订单，并向学生展示支付入口

### Requirement: 学生端不展示平台服务费拆分
系统 MUST 向学生端只展示课节应付总金额，而 MUST NOT 单独展示平台向教师收取的 10% 服务费拆分。

#### Scenario: 学生查看待支付账单
- **WHEN** 学生打开某节课的支付页面
- **THEN** 系统 MUST 仅展示本节应付金额、课程信息和教师信息

### Requirement: 教师端必须展示服务费与到账金额
系统 MUST 在教师账单和钱包页面展示平台服务费、预计到账金额和结算状态。

#### Scenario: 教师查看账单
- **WHEN** 教师打开课程收入或钱包页面
- **THEN** 系统 MUST 展示学生支付金额、平台服务费金额、教师预计到账金额与结算状态

### Requirement: 未支付上一节课时阻塞后续履约
系统 MUST 在学生存在上一节已完成未支付课节时，阻止其预约或进入下一节线上课程。

#### Scenario: 学生尝试预约下一节课
- **WHEN** 学生存在上一节 `COMPLETED_WAIT_PAY` 状态课节
- **THEN** 系统 MUST 阻止其预约新的课节，并提示先完成上一节支付

#### Scenario: 学生尝试进入下一节课堂
- **WHEN** 学生存在上一节 `COMPLETED_WAIT_PAY` 状态课节
- **THEN** 系统 MUST 阻止其进入下一节线上课堂，并提示先完成上一节支付

### Requirement: 支付成功后驱动教师结算
系统 MUST 在课节支付成功后计算平台服务费与教师收入，并将教师收入推进至结算流程。

#### Scenario: 学生支付成功
- **WHEN** 某课节订单支付成功
- **THEN** 系统 MUST 计算 10% 服务费，生成教师收入记录，并更新课节与课程状态
