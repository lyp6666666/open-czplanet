## ADDED Requirements

### Requirement: 用户可查看我发出的申请列表
系统 MUST 支持用户查看“我发出的申请”列表，列表 MUST 仅包含当前用户作为申请发起方的记录，并 MUST 以创建时间倒序展示。

#### Scenario: 查询我发出的申请
- **WHEN** 用户请求“我发出的申请”列表
- **THEN** 系统 MUST 返回发起方为该用户的申请集合，并 MUST 按 createdAt 倒序排序

#### Scenario: 列表支持分页
- **WHEN** 用户携带分页参数请求“我发出的申请”列表
- **THEN** 系统 MUST 按分页参数返回稳定且可续翻的结果

### Requirement: 用户可查看我收到的申请列表
系统 MUST 支持用户查看“我收到的申请”列表，列表 MUST 仅包含当前用户作为申请接收方的记录，并 MUST 以创建时间倒序展示。

#### Scenario: 查询我收到的申请
- **WHEN** 用户请求“我收到的申请”列表
- **THEN** 系统 MUST 返回接收方为该用户的申请集合，并 MUST 按 createdAt 倒序排序

### Requirement: 我收到的申请支持未读标记与未读数
系统 MUST 维护“我收到的申请”的未读状态与未读数，以支持入口提示与列表高亮。

#### Scenario: 新申请到达产生未读
- **WHEN** 有一笔新的申请创建且当前用户为接收方
- **THEN** 系统 MUST 将该申请标记为未读，且 MUST 增加该用户的申请未读数

#### Scenario: 查看详情后清除未读
- **WHEN** 接收方查看一笔未读申请的详情
- **THEN** 系统 MUST 将该申请标记为已读，且 MUST 相应减少该用户的申请未读数

### Requirement: 申请中心可执行通过或拒绝
系统 MUST 支持接收方在申请中心对 `PENDING` 状态的申请执行“通过/拒绝”，并在操作成功后反映到列表与详情。

#### Scenario: 在详情页通过申请
- **WHEN** 接收方在申请详情页点击“通过”
- **THEN** 系统 MUST 将申请状态更新为 `ACCEPTED`，且 MUST 在后续查询中返回更新后的状态

#### Scenario: 在详情页拒绝申请
- **WHEN** 接收方在申请详情页点击“拒绝”
- **THEN** 系统 MUST 将申请状态更新为 `REJECTED`，且 MUST 在后续查询中返回更新后的状态

#### Scenario: 非接收方不可在申请中心处理
- **WHEN** 非接收方尝试在申请中心对申请执行“通过/拒绝”
- **THEN** 系统 MUST 拒绝并返回明确错误
