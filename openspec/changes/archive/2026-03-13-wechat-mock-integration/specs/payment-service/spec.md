## 新增需求 (ADDED Requirements)

### 需求：Mock 支付支持
系统应支持在配置为 Mock 模式时，模拟支付成功流程，而无需真实的微信支付。

#### 场景：生成 Mock 支付参数
- **当** 系统配置为 Mock 模式（`appid=wx_mock_appid`）时
- **且** 请求创建支付订单
- **那么** 系统应生成一个包含 `mock: true` 标记的 `payParams`
- **且** 不应调用真实的微信统一下单接口

#### 场景：前端 Mock 支付回调
- **当** 前端接收到包含 `mock: true` 的支付参数时
- **那么** 前端应直接显示支付成功提示
- **且** 不应调用 `uni.requestPayment`

#### 场景：支付结果通知 Mock
- **当** Mock 支付完成后
- **那么** 后端应能接收到模拟的支付成功通知（或通过内部机制直接标记订单为已支付）
