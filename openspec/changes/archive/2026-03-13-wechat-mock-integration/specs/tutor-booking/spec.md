## 新增需求 (ADDED Requirements)

### 需求：完善的预约流程
系统应提供完整的预约流程，包括选择时间、确认订单，并在订单中关联预约信息。

#### 场景：选择预约时间
- **当** 用户在导师详情页点击“预约”时
- **那么** 系统应弹出时间选择器
- **且** 允许用户输入备注信息

#### 场景：创建关联预约的订单
- **当** 用户确认预约并点击支付时
- **那么** 系统应调用 `TutorAppointmentService` 创建预约
- **且** 调用 `BrokerageOrderService` 创建订单，并关联该预约 ID
- **且** 返回订单 ID 以供支付
