# 提案：微信小程序功能补全

## 为什么 (Why)
当前微信小程序仅实现了基础的“家长找导师”和“预约支付”流程，缺乏家长发布需求、教师找工作、双端角色切换以及聊天沟通等核心功能。为了实现业务闭环并对齐 Web 端体验，需要补全这些缺失的功能模块。

## 变更内容 (What Changes)
- **双端支持**：实现家长端和教师端的角色切换，根据角色展示不同的首页（家长看导师，教师看需求）。
- **需求发布**：允许家长发布家教需求。
- **需求广场**：允许教师查看和筛选家教需求（找工作）。
- **我的管理**：新增“我的需求”和“我的申请”管理页面。
- **聊天功能**：实现基础的聊天列表和聊天室，支持文本消息收发。

## 能力 (Capabilities)

### 新增能力 (New Capabilities)
- `demand-management`: 涵盖需求的发布、编辑、查看（我的需求）以及需求广场的浏览。
- `chat-system`: 涵盖聊天列表、聊天室、消息收发（基于轮询或 WebSocket）。
- `user-role-switch`: 涵盖用户角色的选择、切换以及基于角色的 UI 展示逻辑。

### 修改能力 (Modified Capabilities)
- `wechat-miniprogram-frontend`: 扩展现有的前端架构以支持新的页面和路由。

## 影响 (Impact)
- **前端项目**：`ai-tutor-miniprogram`（新增多个页面和 Store）。
- **后端服务**：无需修改（复用现有的 `ai-tutor-web` API）。
