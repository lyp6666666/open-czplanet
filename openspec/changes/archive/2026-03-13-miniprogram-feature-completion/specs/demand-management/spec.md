## 新增需求 (ADDED Requirements)

### 需求：发布家教需求
系统应允许家长用户发布家教需求。

#### 场景：成功发布需求
- **当** 家长用户填写完整的需求表单并提交时
- **那么** 系统应创建一个新的需求记录
- **且** 跳转到“我的需求”列表页

### 需求：查看需求广场
系统应允许教师用户查看和筛选所有公开的家教需求。

#### 场景：浏览需求列表
- **当** 教师用户进入首页（需求广场）时
- **那么** 系统应显示最新的家教需求列表
- **且** 支持按科目、年级、地区进行筛选

### 需求：管理我的需求
系统应允许家长用户查看和管理自己发布的需求。

#### 场景：查看我的需求
- **当** 家长用户进入“我的需求”页面时
- **那么** 系统应显示该用户发布的所有需求及其状态（如：招募中、已关闭）

### 需求：查看我的申请
系统应允许教师用户查看自己申请过的职位。

#### 场景：查看我的申请记录
- **当** 教师用户进入“我的申请”页面时
- **那么** 系统应显示该用户申请过的所有需求及其状态（如：待沟通、已拒绝、面试中）
