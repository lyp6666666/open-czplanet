## 新增需求 (ADDED Requirements)

### 需求：教师入驻申请
系统应允许未入驻的用户通过小程序提交教师入驻申请。

#### 场景：开始入驻流程
- **当** 用户点击“切换为教师”且当前状态为“未入驻”时
- **那么** 系统应跳转至“教师入驻”页面
- **且** 页面应显示入驻流程步骤条（基本信息 -> 教学偏好 -> 教育背景）

#### 场景：填写基本信息
- **当** 用户填写完基本信息（头像、姓名、简介）并点击下一步时
- **那么** 系统应校验必填项
- **且** 跳转至“教学偏好”步骤

#### 场景：提交入驻申请
- **当** 用户完成所有步骤并点击提交时
- **那么** 系统应调用后端接口提交申请数据
- **且** 成功后跳转至“审核中”页面
- **且** 更新本地用户状态为“审核中”

### 需求：查看入驻状态
系统应根据用户的入驻状态展示不同的页面。

#### 场景：审核中状态
- **当** 用户状态为“审核中”时
- **那么** 教师端首页应显示“您的申请正在审核中”提示
- **且** 仅允许查看状态，不可进行其他操作

#### 场景：审核拒绝状态
- **当** 用户状态为“审核拒绝”时
- **那么** 教师端首页应显示拒绝原因
- **且** 提供“重新申请”按钮，点击后跳转至入驻表单（回填原有数据）
