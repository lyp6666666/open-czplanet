# 提案：微信小程序端教师入驻功能

## 为什么 (Why)
当前微信小程序仅支持简单的角色切换，缺乏完整的教师入驻流程，导致新教师无法直接通过小程序完成注册和资料填写，必须依赖 Web 端操作。为了提升用户体验并实现小程序端的业务闭环，需要在小程序中实现与 Web 端对齐的教师入驻功能。

## 变更内容 (What Changes)
- **入驻流程**：在小程序端实现分步的教师入驻表单（基本信息、教学偏好、教育背景）。
- **状态管理**：处理教师入驻的不同状态（未入驻、审核中、已入驻、审核失败）。
- **API 集成**：复用现有的后端教师入驻接口。

## 能力 (Capabilities)

### 新增能力 (New Capabilities)
- `tutor-onboarding-miniapp`: 涵盖小程序端的教师入驻流程、表单填写与校验、状态展示。

### 修改能力 (Modified Capabilities)
- `user-role-switch`: 修改角色切换逻辑，在切换为教师时检查入驻状态，若未入驻则引导至入驻流程。

## 影响 (Impact)
- **前端项目**：`ai-tutor-miniprogram`（新增入驻页面，修改 `userStore`）。
- **后端服务**：无需修改（复用现有的 `tutor-application-service` API）。
