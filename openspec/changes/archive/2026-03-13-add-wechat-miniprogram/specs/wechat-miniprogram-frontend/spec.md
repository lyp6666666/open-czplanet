## 新增需求（ADDED Requirements）

### 需求：小程序初始化
系统应初始化一个针对微信小程序的 Uni-app 项目，并进行必要的配置。

#### 场景：项目结构创建
- **当** 项目被初始化时
- **那么** 必须存在一个包含 `mp-weixin` 配置（appid）的 `manifest.json` 文件
- **那么** 必须存在一个定义了底部标签栏和初始页面的 `pages.json` 文件

### 需求：底部标签栏导航
小程序应提供原生的底部标签栏用于主要导航。

#### 场景：标签栏显示
- **当** 用户打开小程序时
- **那么** 底部标签栏必须可见
- **那么** 它必须包含“首页”、“消息”和“我的”标签
- **那么** 点击标签必须切换视图到相应的页面

### 需求：首页
首页应显示可用导师列表和搜索功能。

#### 场景：查看导师列表
- **当** 用户查看首页时
- **那么** 必须显示导师列表
- **那么** 每个导师卡片必须显示姓名、科目和评分

### 需求：用户个人资料页
我的页面（用户个人资料）应显示用户信息，如果未认证则允许登录。

#### 场景：未认证状态
- **当** 未认证用户查看我的页面时
- **那么** 必须显示“登录”按钮
- **那么** 点击“登录”必须触发微信登录流程

#### 场景：已认证状态
- **当** 已认证用户查看我的页面时
- **那么** 必须显示用户的昵称和头像
- **那么** 必须提供“退出登录”按钮
