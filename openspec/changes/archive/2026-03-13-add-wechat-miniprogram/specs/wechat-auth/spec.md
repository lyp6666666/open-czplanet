## 新增需求（ADDED Requirements）

### 需求：微信 Code 交换
系统应提供一个 API 端点，用于将微信小程序登录 code 交换为会话。

#### 场景：成功的 Code 交换
- **当** 后端接收到有效的微信 `jscode` 时
- **那么** 它应调用微信 `jscode2session` API
- **那么** 它应检索 `openid` 和 `session_key`
- **那么** 它应返回一个包含与该 `openid` 关联的 JWT 令牌的成功响应

#### 场景：无效的 Code
- **当** 后端接收到无效或过期的微信 `jscode` 时
- **那么** 它应返回一个错误响应

### 需求：用户绑定
系统应将微信 `openid` 绑定到现有的用户账户，或者如果不存在则创建一个新用户。

#### 场景：现有用户登录
- **当** `openid` 匹配现有的用户记录时
- **那么** 系统应为该用户生成一个 JWT
- **那么** 响应应包含用户的个人资料信息

#### 场景：新用户注册
- **当** `openid` 不匹配任何现有用户时
- **那么** 系统应创建一个带有该 `openid` 的新用户记录
- **那么** 系统应为新用户生成一个 JWT
- **那么** 响应应指示已创建新用户（以便前端在需要时提示更多详细信息）
