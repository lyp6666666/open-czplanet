## ADDED Requirements

### Requirement: 聊天输入框上方提供“发起合作”入口
系统 MUST 在聊天页面的聊天输入框上方提供“发起合作”入口，用于双方沟通达成一致后发起家教合作提案。

#### Scenario: 发起方可见入口
- **WHEN** 用户进入任意可聊天的 1v1 会话页面
- **THEN** 系统 MUST 展示“发起合作”入口

### Requirement: 发起合作时需填写必要信息
系统 MUST 在发起合作时要求发起方填写必要信息，并在提交前进行校验。必要信息 MUST 至少包含：收费标准、上课时间、上课频次。

#### Scenario: 必填字段校验失败
- **WHEN** 用户发起合作但缺少任一必填字段
- **THEN** 系统 MUST 阻止提交并提示缺失项

#### Scenario: 必填字段校验成功
- **WHEN** 用户发起合作且必填字段均已填写且通过校验
- **THEN** 系统 MUST 允许提交并生成一条合作提案

### Requirement: 合作提案以结构化消息块形式发送到聊天中
系统 MUST 在聊天中以结构化消息块的形式发送合作提案，并在消息块中展示提案关键信息与当前状态。

#### Scenario: 发送合作提案消息块
- **WHEN** 发起方提交合作提案
- **THEN** 系统 MUST 在该会话中生成并发送一条合作提案消息块，且双方 MUST 可见

### Requirement: 接收方可同意或拒绝合作提案
系统 MUST 允许接收方在合作提案消息块内选择同意或拒绝，且该操作 MUST 产生可追溯的状态变更记录。

#### Scenario: 接收方同意合作提案
- **WHEN** 接收方在合作提案消息块中点击“同意”
- **THEN** 系统 MUST 将合作提案状态更新为“已同意”，并记录操作者与时间

#### Scenario: 接收方拒绝合作提案
- **WHEN** 接收方在合作提案消息块中点击“拒绝”
- **THEN** 系统 MUST 将合作提案状态更新为“已拒绝”，并记录操作者与时间

### Requirement: 合作提案状态变更对双方可见
系统 MUST 确保合作提案的最新状态对会话双方可见，并且 MUST 在页面刷新或重新进入会话后仍可正确展示状态。

#### Scenario: 刷新后状态仍正确
- **WHEN** 任一方刷新聊天页面或重新进入该会话
- **THEN** 系统 MUST 展示合作提案的最新状态

### Requirement: 合作提案记录用于后续中介费结算依据
系统 MUST 保留合作提案的内容与状态流转记录，且 MUST 能被后续结算流程引用以计算与收取平台中介费。

#### Scenario: 可按会话查询合作提案
- **WHEN** 结算流程需要查询某会话中的合作提案记录
- **THEN** 系统 MUST 能够按会话标识检索合作提案及其状态流转信息
