## ADDED Requirements

### Requirement: 聊天消息展示发送方头像与姓名
系统 MUST 在聊天消息列表中展示每条消息的发送方头像与姓名，并以“自己/对方”的对话样式呈现（类似微信的左右布局）。

#### Scenario: 渲染自己发送的消息
- **WHEN** 用户在聊天页查看一条由自己发送的消息
- **THEN** 系统 MUST 在该消息行展示“本人头像与姓名”，并以“我方”样式对齐

#### Scenario: 渲染对方发送的消息
- **WHEN** 用户在聊天页查看一条由对方发送的消息
- **THEN** 系统 MUST 在该消息行展示“对方头像与姓名”，并以“对方”样式对齐

### Requirement: 点击对方头像打开名片弹窗
系统 MUST 支持用户在聊天页面点击对方头像打开名片弹窗。名片弹窗 MUST 以可关闭的小窗口形式展示，并且 MUST 在不离开当前聊天页的情况下完成查看与关闭。

#### Scenario: 点击头像打开名片
- **WHEN** 用户在聊天页点击对方头像
- **THEN** 系统 MUST 打开名片弹窗并展示对方基础信息

#### Scenario: 关闭名片弹窗
- **WHEN** 用户点击名片弹窗的关闭按钮
- **THEN** 系统 MUST 关闭名片弹窗且聊天页面状态 MUST 保持不变

### Requirement: 名片内容按用户角色展示
系统 MUST 根据被查看用户的角色展示不同的名片内容：若为教师，则展示教师基础信息与基本介绍；若为学生，则展示学生基础信息与岗位相关信息。

#### Scenario: 查看教师名片
- **WHEN** 用户查看对方名片且对方为教师
- **THEN** 名片弹窗 MUST 展示教师的基础信息与基本介绍

#### Scenario: 查看学生名片
- **WHEN** 用户查看对方名片且对方为学生
- **THEN** 名片弹窗 MUST 展示学生的基础信息与岗位相关信息

### Requirement: 名片信息需受权限与可见性约束
系统 MUST 对名片展示内容施加权限与可见性约束，且 MUST 确保名片接口不返回不应对对端暴露的敏感字段。

#### Scenario: 请求名片时的鉴权
- **WHEN** 未登录用户尝试请求名片信息
- **THEN** 系统 MUST 拒绝请求
