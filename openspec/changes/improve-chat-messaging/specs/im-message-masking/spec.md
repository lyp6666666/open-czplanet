## ADDED Requirements

### Requirement: IM 文本消息敏感信息脱敏展示
系统 MUST 对 IM 文本消息中的敏感联系方式信息进行识别与脱敏展示。被识别命中的敏感片段 MUST 在展示与下发给客户端时替换为 `***`，且替换后文本 MUST 保持可读（非命中片段保持原样）。

#### Scenario: 发送包含手机号的文本消息
- **WHEN** 用户发送一条文本消息，内容包含可识别为手机号的片段
- **THEN** 系统 MUST 将该消息保存为文本消息，并在客户端展示的内容中将手机号片段替换为 `***`

#### Scenario: 发送包含微信标识的文本消息
- **WHEN** 用户发送一条文本消息，内容包含可识别为微信/微信号的片段（含常见变体，如 wx、wxid、微信等）
- **THEN** 系统 MUST 在客户端展示的内容中将命中的敏感片段替换为 `***`

#### Scenario: 发送包含 QQ 标识的文本消息
- **WHEN** 用户发送一条文本消息，内容包含可识别为 QQ/QQ 号的片段（含常见变体）
- **THEN** 系统 MUST 在客户端展示的内容中将命中的敏感片段替换为 `***`

### Requirement: 消息落库存原文并记录屏蔽标记
系统 MUST 在数据库中保留用户发送的原始文本内容用于风控与追溯，并 MUST 记录该条消息是否命中屏蔽规则的标记字段。

#### Scenario: 命中规则的消息落库
- **WHEN** 用户发送的文本消息命中任一敏感信息规则
- **THEN** 系统 MUST 将原始文本保存到消息存储中，且 MUST 将该条消息标记为“被屏蔽”

#### Scenario: 未命中规则的消息落库
- **WHEN** 用户发送的文本消息未命中任何敏感信息规则
- **THEN** 系统 MUST 将该条消息标记为“未被屏蔽”

### Requirement: 脱敏对发送回包、历史拉取与实时推送一致生效
系统 MUST 在以下链路中一致应用脱敏展示逻辑：发送消息成功后的返回、消息分页拉取接口返回、实时推送通道下发的数据。

#### Scenario: 发送回包脱敏
- **WHEN** 用户发送一条命中规则的文本消息并收到发送成功返回
- **THEN** 返回的消息体中展示内容 MUST 已被替换为 `***`

#### Scenario: 历史拉取脱敏
- **WHEN** 客户端通过分页接口拉取包含命中规则的历史文本消息
- **THEN** 返回的消息体中展示内容 MUST 已被替换为 `***`

#### Scenario: 实时推送脱敏
- **WHEN** 客户端通过实时推送通道接收一条命中规则的文本消息
- **THEN** 推送的消息体中展示内容 MUST 已被替换为 `***`

### Requirement: 客户端可识别消息是否被屏蔽
系统 MUST 在消息下发的结构化消息体中提供该消息是否被屏蔽的标识，以便客户端展示提示或样式差异。

#### Scenario: 下发被屏蔽标记
- **WHEN** 客户端接收到一条命中规则的文本消息
- **THEN** 消息体 MUST 包含“已屏蔽”的标识字段，且该标识 MUST 为 true
