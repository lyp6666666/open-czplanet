## ADDED Requirements

### Requirement: 机构可以创建需求并保存草稿或发布
系统 MUST 允许机构创建需求单，并支持保存为草稿或直接发布。系统 MUST 对需求字段进行校验（例如标题、科目/年级、地点/线上、期望上课时间、预算区间、补充说明等），校验失败 MUST 返回明确错误。

#### Scenario: 机构保存草稿
- **WHEN** 机构填写部分字段并点击保存草稿
- **THEN** 系统保存草稿并允许后续继续编辑

#### Scenario: 机构发布需求成功
- **WHEN** 机构填写必填字段并点击发布
- **THEN** 系统将需求状态置为已发布并使其在教师侧可见

### Requirement: 机构可编辑未进入履约阶段的需求
系统 MUST 允许机构编辑草稿与已发布但未进入履约阶段的需求。系统 MUST 在需求已进入履约阶段后限制关键字段修改以避免争议（例如预算、地点、时间等），并提供变更记录。

#### Scenario: 已发布但未确认教师前可编辑
- **WHEN** 机构编辑已发布需求并提交更新
- **THEN** 系统保存更新并在教师侧展示最新信息

#### Scenario: 进入履约阶段后限制关键字段修改
- **WHEN** 机构尝试修改已进入履约阶段需求的预算字段
- **THEN** 系统拒绝修改并提示需通过客服或协商流程处理

### Requirement: 机构可下架或关闭未进入履约阶段的需求
系统 MUST 允许机构下架/关闭草稿或已发布但未进入履约阶段的需求。系统 MUST 明确区分“机构主动关闭”和“系统风控关闭”，并在教师侧同步可见性变化。

#### Scenario: 机构下架已发布需求
- **WHEN** 机构对已发布需求执行下架
- **THEN** 系统将需求标记为已关闭且教师侧不再可申请

### Requirement: 需求发布合规提示与风险前置
系统 MUST 在机构发布需求时展示合规提示（例如不得发布违法/涉黄赌毒/虚假信息）。系统 SHOULD 支持对高风险内容触发人工审核或自动拦截，并在管理端可追溯。

#### Scenario: 发布内容触发拦截
- **WHEN** 机构发布需求内容命中高风险规则
- **THEN** 系统拒绝发布并提示需要修改或等待审核
