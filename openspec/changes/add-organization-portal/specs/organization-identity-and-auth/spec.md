## ADDED Requirements

### Requirement: 机构账号仅可由管理端创建与发放
系统 MUST 禁止任何非管理员主体通过公开注册流程创建机构账号。系统 MUST 仅允许具有管理员权限的主体创建机构账号并发放初始登录凭证。

#### Scenario: 学生尝试创建机构账号
- **WHEN** 已登录学生调用创建机构账号的接口或页面入口
- **THEN** 系统拒绝请求并返回无权限错误

#### Scenario: 管理员创建机构账号并生成初始凭证
- **WHEN** 管理员在管理端提交机构名称与登录账号信息并点击创建
- **THEN** 系统创建机构主体与机构登录账号，并生成一次性的初始登录凭证可供客服转交

### Requirement: 机构登录入口与学生/教师登录入口隔离
系统 MUST 提供独立的机构登录入口，并在交互与接口层确保机构登录不等同于学生/教师登录入口的自助注册与登录流程。系统 MUST 防止机构身份通过学生/教师登录入口被获取。

#### Scenario: 机构用户通过机构登录入口完成登录
- **WHEN** 机构用户在机构登录入口提交有效凭证
- **THEN** 系统签发会话令牌并将用户身份标记为机构身份

#### Scenario: 机构用户尝试通过学生/教师登录入口获取机构身份
- **WHEN** 机构用户在学生/教师登录入口尝试登录或注册
- **THEN** 系统不授予机构身份能力且不暴露机构专用接口

### Requirement: 机构会话令牌携带机构归属信息
系统 MUST 在登录会话中绑定机构主体标识（organizationId）与身份类型（ORG），并在鉴权时使用该归属信息做资源访问控制。系统 MUST 禁止跨机构访问。

#### Scenario: 机构访问自身资源成功
- **WHEN** 机构使用有效令牌访问其名下需求列表
- **THEN** 系统仅返回该机构归属的需求数据

#### Scenario: 机构访问其他机构资源被拒绝
- **WHEN** 机构尝试访问其他机构的需求详情或财务明细
- **THEN** 系统拒绝请求并返回无权限错误

### Requirement: 机构账号安全控制（强制改密/重置/冻结）
系统 MUST 支持机构账号首次登录强制改密。系统 MUST 支持管理员或客服发起密码重置并生成新的临时凭证。系统 MUST 支持冻结/禁用机构账号，冻结后所有机构侧写操作与敏感读操作 MUST 被拒绝。

#### Scenario: 首次登录触发强制改密
- **WHEN** 机构使用初始凭证首次登录成功
- **THEN** 系统要求完成密码更新后才允许进入机构工作台

#### Scenario: 冻结后阻断机构侧操作
- **WHEN** 管理员冻结机构账号
- **THEN** 机构再次发起发布需求或查看分账明细的请求被拒绝

### Requirement: 首期不支持机构子账号
系统 MUST 不提供机构子账号/员工账号创建能力。系统 MUST 以“机构主账号”为最小可用形态完成机构端闭环。

#### Scenario: 机构尝试创建子账号
- **WHEN** 机构在机构端尝试创建员工账号
- **THEN** 系统不提供该入口或拒绝请求并提示暂不支持

### Requirement: 鉴权事件可审计
系统 MUST 记录机构账号的关键鉴权事件审计日志（登录成功/失败、改密、重置、冻结/解冻、异常频控触发），并支持按机构查询与导出（至少面向管理端可见）。

#### Scenario: 登录失败被记录
- **WHEN** 机构使用错误凭证登录
- **THEN** 系统记录失败原因与时间，并对重复失败触发频控策略
