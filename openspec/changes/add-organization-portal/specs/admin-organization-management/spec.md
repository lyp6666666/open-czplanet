## ADDED Requirements

### Requirement: 管理员可以创建与维护机构主体信息
系统 MUST 允许管理员创建机构主体并维护机构资料（名称、简介、联系人、联系方式、地址、资质材料、状态）。系统 MUST 支持将机构状态设置为正常/冻结/禁用，并对前台展示与机构侧权限产生影响。

#### Scenario: 管理员创建机构并提交资料
- **WHEN** 管理员在机构管理页面填写机构资料并提交创建
- **THEN** 系统创建机构主体并返回机构标识

#### Scenario: 管理员禁用机构后影响前台能力
- **WHEN** 管理员将机构状态设为禁用
- **THEN** 机构无法登录或执行敏感操作，且机构发布中的需求可按策略下架或标记风险提示

### Requirement: 管理端创建机构登录账号并生成发放凭证
系统 MUST 支持管理端为机构创建登录账号并生成可发放的初始凭证。系统 MUST 支持对机构账号执行重置密码与强制改密策略。

#### Scenario: 生成初始凭证用于客服发放
- **WHEN** 管理员为机构创建登录账号并选择生成初始凭证
- **THEN** 系统生成一次性凭证并在管理端可见，便于客服安全转交

### Requirement: 管理端可查看机构发布单子及其状态分布
系统 MUST 允许管理员查看某机构发布的单子列表，并能按状态筛选与统计（草稿、已发布、申请中、待支付、已支付可聊、进行中、已完成、已关闭等）。系统 MUST 支持点击进入单子详情查看完整流转与关键事件。

#### Scenario: 管理员查看机构单子状态统计
- **WHEN** 管理员进入机构详情并打开单子统计
- **THEN** 系统展示各状态数量与可下钻列表

### Requirement: 管理端可查看与处置机构资金相关风险
系统 MUST 允许管理员查看机构的分账台账汇总与明细，并支持执行风险处置动作（冻结结算、解冻、冲正、强制退款等，按权限控制）。系统 MUST 记录所有资金处置操作的审计日志。

#### Scenario: 管理员冻结机构结算
- **WHEN** 管理员对机构执行冻结结算操作
- **THEN** 系统将该机构相关台账置为冻结或禁止进入结算，并记录操作日志
