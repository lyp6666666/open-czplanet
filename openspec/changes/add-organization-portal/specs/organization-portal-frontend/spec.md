## ADDED Requirements

### Requirement: 机构工作台提供与学生端一致的核心信息架构
系统 MUST 为机构提供工作台首页与导航，覆盖发布需求、单子管理、申请处理、消息通知与机构资料等核心入口。系统 MUST 复用学生端能力与信息结构，使机构用户能在低学习成本下完成发布与管理。

#### Scenario: 机构登录后进入工作台首页
- **WHEN** 机构用户完成登录
- **THEN** 系统展示机构工作台首页，并提供发布需求与查看单子列表的入口

### Requirement: 机构主页可被教师端访问并展示机构信息
系统 MUST 提供可公开访问的机构主页，用于教师在浏览需求时了解发布方。机构主页 MUST 展示机构名称、简介、资质信息（若已配置）、服务范围与投诉/联系客服入口。

#### Scenario: 教师从需求详情进入机构主页
- **WHEN** 教师在需求详情点击发布方机构名称
- **THEN** 系统打开机构主页并展示机构基础信息与相关入口

### Requirement: 需求列表与详情展示发布者类型为机构
系统 MUST 在教师端需求列表与详情中展示发布者类型（学生/机构）。当发布者为机构时，系统 MUST 展示机构名称并提供跳转到机构主页的入口。

#### Scenario: 教师浏览需求列表识别机构单
- **WHEN** 教师打开需求列表
- **THEN** 系统在机构发布的需求条目上展示机构名称与“机构发布”标识

### Requirement: 机构单展示责任划分说明与平台承诺提示
系统 MUST 在教师端的机构单列表与详情提供清晰的责任划分说明与平台承诺提示（例如提示卡/帮助入口）。提示内容 MUST 明确机构为需求发布与履约主体，平台提供信息撮合、支付托管与纠纷介入机制，但不保证授课质量与履约结果。

#### Scenario: 教师查看机构单详情看到责任说明
- **WHEN** 教师打开机构发布的需求详情页
- **THEN** 系统展示“机构单”标识与责任说明提示，并提供投诉/联系客服入口

### Requirement: 机构可在前端查看其发布的单子状态与关键动作
系统 MUST 允许机构在工作台查看其发布单子的状态（草稿、已发布、进行中、已完成、已取消/关闭等）与关键动作（编辑、下架、处理申请、查看聊天入口、查看信息费与分账摘要）。

#### Scenario: 机构查看单子列表并筛选状态
- **WHEN** 机构打开单子管理并选择状态筛选条件
- **THEN** 系统仅展示匹配状态的单子并显示每单的当前状态与可执行动作

### Requirement: 机构端入口与学生/教师端入口互不混淆
系统 MUST 在前端提供明确的入口隔离，使普通学生用户无法通过导航或入口切换到机构工作台。系统 MUST 在未授权访问机构域页面时强制跳转到机构登录入口。

#### Scenario: 学生尝试访问机构域页面
- **WHEN** 学生用户直接访问机构域页面地址
- **THEN** 系统跳转到机构登录入口并要求使用机构凭证登录
