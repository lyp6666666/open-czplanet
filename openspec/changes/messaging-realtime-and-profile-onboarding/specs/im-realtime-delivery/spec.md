## ADDED Requirements

### Requirement: 收件方无需刷新即可看到新消息
当发送方在会话中发送消息后，收件方客户端 MUST 通过服务端推送通道在不刷新页面的情况下收到该消息事件，并在 UI 上主动渲染。

#### Scenario: 收件方在会话页实时追加
- **WHEN** 收件方已打开会话房间页且保持在线，发送方向该房间发送一条新消息
- **THEN** 收件方会话页 MUST 立即追加该消息并展示在消息列表末尾

### Requirement: 消息列表实时更新预览与排序
当收件方不在当前会话页（例如停留在会话列表页或其他页面）时，系统 MUST 更新未读数；若用户位于会话列表页，列表 MUST 主动更新对应会话的最后一条消息预览与时间，并将该会话移动到列表顶部（与常见 IM 一致）。

#### Scenario: 会话列表无需刷新即可更新
- **WHEN** 收件方停留在会话列表页，发送方向某会话发送一条新消息
- **THEN** 该会话在列表中的预览内容/时间 MUST 更新且会话排序 MUST 提升到顶部，同时该会话未读红点 MUST 增加

### Requirement: 实时事件必须去重
客户端在网络抖动、断线重连或多源触发（例如列表回源 + 推送）情况下 MAY 收到重复事件；客户端 MUST 基于消息 id 对渲染与未读累计做去重，避免同一条消息导致重复展示或重复累加未读数。

#### Scenario: 断线重连不重复追加
- **WHEN** 客户端断线后重连并收到重复的同一条 message 事件（msgId 相同）
- **THEN** 客户端 MUST 只渲染一次该消息且未读数 MUST 只累计一次

### Requirement: 推送连接重建后需要校准
客户端在重连成功后 MUST 触发一次“未读与会话列表校准”（例如刷新未读汇总或拉取会话列表首屏），以消除断线期间的状态偏差。

#### Scenario: 重连后未读恢复正确
- **WHEN** 客户端断线期间收到多条离线消息，随后恢复网络并重连成功
- **THEN** 客户端 MUST 在校准后展示正确的会话未读数与顶部未读汇总
