## ADDED Requirements

### Requirement: 家教首次入驻必须完成第一步基础信息
当用户首次以“当家教”身份使用产品时，系统 MUST 引导用户进入“两步入驻流程”；第一步（头像上传、真实姓名）为必填项，未完成前 MUST 不允许进入需要“家教身份”的核心页面。

#### Scenario: 首次进入家教端被引导到第一步
- **WHEN** 用户以家教身份登录且服务端未标记基础信息已完成
- **THEN** 前端 MUST 跳转到入驻第一步页面，并在提交成功前阻止访问家教端核心功能页面

### Requirement: 入驻第一步写入头像与真实姓名并可回显
系统 MUST 支持上传/选择头像并保存可访问的头像 URL；系统 MUST 保存真实姓名并在后续“简历/我的”页面与入驻流程中回显。

#### Scenario: 第一部提交后回显一致
- **WHEN** 用户在第一步提交头像与真实姓名成功
- **THEN** 刷新页面后再次进入“简历/我的”或入驻流程，头像与真实姓名 MUST 正确回显

### Requirement: 入驻第二步完善简历信息并支持跳过
入驻第二步 MUST 支持分步填写简历信息：学历、所在城市、最高学历学校、自我介绍、擅长学科（多选）、支持教学方式（线上/线下/均可）。第二步 MUST 提供“跳过”按钮；用户选择跳过后 MUST 可以继续使用产品，且后续仍可在“简历/我的”页面补齐并保存这些信息。

#### Scenario: 第二步可跳过且不阻塞使用
- **WHEN** 用户完成第一步后进入第二步并点击“跳过”
- **THEN** 系统 MUST 结束入驻引导并允许用户进入家教端正常页面，且第二步字段保持未填写/默认状态

### Requirement: 擅长学科支持多选并可稳定存储与回显
系统 MUST 支持用户选择多个擅长学科，并确保保存与回显一致；后端存储格式可以是兼容现有字段的可解析字符串或结构化字段，但对外接口 MUST 能表达多选语义。

#### Scenario: 多选学科保存后仍为多选
- **WHEN** 用户在第二步选择多个学科并保存成功
- **THEN** 再次打开简历编辑页时 MUST 回显相同的多选结果

### Requirement: 教学方式必须为受控枚举
系统 MUST 将支持教学方式限定为受控枚举值：线上教学、线下教学、均可，并在保存与展示时保持一致的语义映射。

#### Scenario: 教学方式枚举保存与展示一致
- **WHEN** 用户选择“线上教学”并保存成功
- **THEN** 简历展示页与再次编辑时 MUST 显示为“线上教学”
