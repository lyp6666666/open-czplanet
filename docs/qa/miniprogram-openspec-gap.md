# 微信小程序 OpenSpec 实现差距跟踪

更新时间：2026-05-01

## 已完成到可验收的部分

- 页面骨架已覆盖：
  - `我的合作`
  - `合作详情`
  - `单节课详情`
  - `课程表`
  - `课堂准备`
  - `课堂承接页`
  - `课堂 web-view`
  - `课后 AI 总结`
- 真实课堂承接已接通：
  - 小程序 `prepare -> joinToken`
  - H5 桥接页透传真实 token
  - Web 课堂页优先消费预签发 token
- 课堂返回刷新已补齐：
  - 从课堂返回后，合作详情 / 课节详情 / 课程表会自动刷新
- 构建门禁已补齐一部分：
  - `npm run type-check`
  - `npm run build:mp-weixin`
  - 生产构建禁止 `localhost/127.0.0.1` Base URL

## 仍未完全闭环的 OpenSpec 项

### 1. 页面架构与工程基础

- `pages.json` 还未做分包优化与页面地图文档。
- API/类型层虽然已覆盖课程、直播、课表、聊天等核心域，但还没有形成完整接口映射文档。
- 统一组件层仍偏分散，底部操作栏、权限提示、业务卡片还没有完全沉淀成统一组件。

### 2. 用户上下文与角色守卫

- 登录前回跳虽已覆盖多处，但还没有针对所有课程/支付/课堂入口做系统验收记录。
- 退出登录对聊天轮询、支付轮询、未读聚合的清理闭环还缺专门验证。

### 3. 学生端实现

- 学生首页 / 找老师 / 发布需求 / 我的需求已基本可用，但还没有补“页面到真实接口”的统一验收文档。
- 线下需求主流程尚未完成一轮真实联调闭环记录。

### 4. 教师端实现

- 教师入驻、状态页、需求广场、详情页已有实现，但还未完成从入驻到发起申请的整链验收记录。
- 钱包/结算入口仍属于预留，不算闭环完成。

### 5. 申请、支付与聊天闭环

- 真实支付环境闭环未完成最终验证记录：
  - openid 绑定
  - JSAPI 参数
  - 微信合法域名
- 聊天“输入中 / 送达 / 已读”是否完全与后端一致，还缺真实环境验收记录。
- 合作提案从聊天发起到课程生成，仍缺一轮面向小程序的完整测试报告。

### 6. 课程、课节、课程表与课堂

- 已有页面与主要动作，但还缺以下真实闭环验收：
  - 新增课节
  - 调课
  - 确认改期
  - 取消课节
  - 结课
  - 课后 AI 总结状态切换
- 线下试课退款虽然前端校验已补强，但仍缺真实提交流程记录。
- 原生小程序课堂未实现，当前仍采用 H5 承接方案。

### 7. 真实接口联调

- 缺少系统化联调结果沉淀：
  - 测试学生账号
  - 测试教师账号
  - 需求 ID
  - 申请 ID
  - 订单号
  - 房间 ID
  - 课程 ID
  - 课节 ID
- 线下需求与线下退款分支尚未形成闭环记录。

### 8. 测试与发布门禁

- 尚未完成：
  - 微信开发者工具启动检查记录
  - 真机验证记录
  - 支付取消/失败/超时异常记录
  - 聊天未解锁异常记录
  - 申请拒绝 / 合作拒绝 / 课节异常记录
  - AppID / 合法域名 / 隐私协议 / Mock 开关 / 包体积的发布前检查单

2. 形成页面到接口映射表
