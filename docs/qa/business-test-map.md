# AI Tutor 业务测试地图

## 1. 测试环境

QA 测试机固定为 `117.72.111.39`。

建议统一入口与配置口径：

| 类型 | 建议值 | 说明 |
| --- | --- | --- |
| API 网关 | `http://117.72.111.39:18080` | 自动化默认 `QA_API_BASE_URL` |
| Web 前台 | `http://117.72.111.39:5173` | 自动化默认 `QA_WEB_BASE_URL` |
| 管理端 | `http://117.72.111.39:5174` | 若端口不同，以 Nginx/部署脚本为准 |
| 数据库 | `117.72.111.39:3306/ai_tutor` | 仅测试环境允许导入 QA seed |
| Redis/MinIO/MQ | 同测试机部署 | 由测试机 docker compose 或远程服务维护 |

测试环境原则：

- 使用 `qa` profile，允许测试辅助接口，例如验证码读取、支付签名模拟、测试数据清理。
- 禁止连接真实短信、真实支付扣款、生产 MinIO bucket。
- 自动化数据全部使用 `QA_` 前缀或 `qa-` 幂等键，便于查询和清理。
- 每次发布前至少执行 API smoke；发布窗口执行 regression；线上仅做只读或低风险 smoke。

## 2. 业务质量主线

本项目的质量保障按交易撮合闭环设计：

`登录/资料 -> 需求/教师供给 -> 申请 -> 信息费支付 -> 聊天解锁 -> 试听/正式课 -> 课节支付 -> 课堂/AI -> 退款/结算 -> 后台运营`

优先级定义：

| 优先级 | 定义 | 典型场景 |
| --- | --- | --- |
| P0 | 影响资金、聊天准入、履约状态一致性 | 支付回调、退款、聊天解锁、课程状态 |
| P1 | 影响主流程转化 | 登录、发需求、申请、IM 消息、预约 |
| P2 | 影响运营效率或体验 | 后台审核、首页配置、收藏、搜索 |
| P3 | 边缘能力与展示 | 邮件通知、邀请返利、AI 摘要展示 |

## 3. 业务测试地图

| 业务域 | 核心对象 | P0/P1 风险 | 关键用例 | 自动化层级 |
| --- | --- | --- | --- | --- |
| 账号与角色 | `user`, JWT, admin user | 登录失败、角色越权、token 失效 | 手机号登录、验证码错误、学生/老师/机构/管理员访问边界 | API smoke + UI smoke |
| 资料与认证 | `student_profile`, `teacher_profile`, MinIO | 资料缺失影响撮合，认证状态错 | 学生资料保存、老师简历、实名认证待审/通过/驳回、头像上传 | API regression + UI smoke |
| 科目与教师供给 | `position_post`, `teacher_job_posting` | 搜索不准、下架仍展示 | 科目树、老师服务上架/下架、城市/线上筛选、价格排序 | API regression |
| 需求发布与广场 | `student_job_posting` | 待审/拒绝/关闭状态错，越权修改 | 家长发需求、学生自发需求、机构单、待审核、拒绝、关闭、搜索分页 | API smoke + UI smoke |
| 收藏与浏览 | `parent_favorite_tutor`, `tutor_favorite_demand` | 重复收藏、取消后状态错 | 收藏需求、收藏老师、重复收藏幂等、列表回显 | API regression |
| 申请流 | `tutor_application` | 申请重复、接受后状态不一致 | 老师申请需求、学生申请老师、待处理、接受、拒绝、已读 | API smoke |
| 信息费/聊天解锁 | `brokerage_order`, `payment_order`, `room` | 付费未解锁、重复解锁、未付费可聊天 | 申请接受后待支付、支付成功后 `CHAT_ENABLED`、重复回调幂等 | API P0 + E2E |
| IM 聊天 | `room`, `message`, `room_read_state`, `chat_realtime_event` | 消息丢失、顺序错、SSE 不推送、已读未读错 | 双端发消息、图片/文本/系统消息、SSE、已读、补偿事件 | API + UI + E2E |
| 合作提案 | `collaboration_proposal` | 过期/接受/拒绝状态错 | 老师发提案、学生接受、拒绝、幂等请求 | API regression |
| 试听与课程 | `course_enrollment`, `tutor_appointment` | 课程状态错、过期未提醒、试课退款错 | 试课中、正式课表待提交、教学中、已完成、取消/改期 | API regression + E2E |
| 课节支付与结算 | `lesson_payment_order`, `teacher_settlement` | 收款/结算金额错 | 课节待支付、支付成功、平台服务费、老师可结算 | API P0 |
| 退款 | `refund_request`, `payment_refund` | 重复退款、金额错、审核状态错 | 聊天信息费退款、试课退款、审核通过/拒绝、原路退款失败 | API P0 + admin UI |
| 直播课堂 | `live_class_session`, participants/events | 进房失败、课堂状态错、断线事件丢失 | 待上课、进行中、已结束、设备检测、事件流水 | API + Playwright |
| AI 课堂 | `ai_task`, `ai_lesson_report`, transcript | AI 失败影响课后报告 | 任务生成中/完成/失败、转写、阶段摘要、课后报告 | API regression |
| 邮件通知 | email tables | 验证/通知误发或漏发 | 绑定邮箱、课后总结通知、失败重试、退信状态 | API regression |
| 邀请推广 | invite tables | 返利错、重复绑定、防刷 | 邀请码、邀请关系、奖励冻结/可结算/已支付 | API regression |
| 后台运营 | admin controllers | 越权、误审、配置不生效 | 登录、老师认证审核、需求审核、退款审核、首页轮播 | API + admin UI |

## 4. Smoke 集合

PR 或测试机部署后必须跑：

| 编号 | 链路 | 断言 |
| --- | --- | --- |
| S1 | 学生登录 -> 查看我的资料 | token 有效，资料返回正确 |
| S2 | 老师登录 -> 浏览需求 -> 收藏需求 | 列表可用，收藏幂等 |
| S3 | 老师申请需求 -> 学生看到申请 | 申请状态 `PENDING`，接收方可见 |
| S4 | 学生接受申请 -> 生成信息费订单 | 申请 `ACCEPTED`，聊天准入 `PAYMENT_REQUIRED` |
| S5 | 模拟支付成功 -> 聊天解锁 -> 双方发消息 | 订单 `SUCCESS/PAID`，申请 `CHAT_ENABLED`，消息落库 |
| S6 | 管理员登录 -> 查看退款/审核列表 | 后台接口可用，权限正确 |

## 5. Regression 集合

发布前或夜间跑：

- 账号：验证码错误、过期 token、角色越权、被禁用账号。
- 需求：待审核、审核拒绝、已关闭、机构单、学生自发单、线上/线下/both。
- 申请：重复申请、已读未读、接受、拒绝、接受后重复处理。
- 支付：订单过期、重复回调、签名错误、金额不一致、事件投递失败补偿。
- IM：消息分页、SSE 推送、已读状态、图片消息、撤回消息、关闭房间不可发。
- 课程：试听中、教学中、已完成、取消、改期、正式课表 deadline。
- 退款：聊天退款 100%、试课退款 80%、审核拒绝、退款失败重试。
- 直播：未到入房时间、老师先入房、学生先入房、课堂结束、webhook 幂等。
- 后台：老师认证通过/驳回、首页轮播配置、退款审核、需求审核。

## 7. 准入准出

- QA seed 可重复导入且不报错。
- API 网关健康检查通过。
- `qa/automation` smoke 可连通 117 测试机。
