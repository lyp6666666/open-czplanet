## 2. 前端目标

本次前端改动要让用户清楚理解：

1. 主邮箱用于未读消息提醒、开课提醒、课后总结。
2. 学生可额外添加一个“课后总结邮箱”，适合填写家长邮箱。
3. 第二邮箱只接收课后总结，不影响登录，也不接收开课提醒和消息提醒。
4. 未绑定邮箱时，在聊天、课程、总结等关键场景给出适度提示。

## 3. 信息架构变动

### 3.1 新增页面

建议新增独立页面：

1. Web：`/account/email`
2. 小程序：`pages/account/email`

页面名称：

`邮箱提醒设置`

### 3.2 入口位置

1. 我的页面：资料/账号设置区域新增“邮箱提醒”
2. 聊天列表页：未绑定主邮箱时展示提示条
3. 聊天详情页：未绑定主邮箱时可展示顶部轻提示
4. 我的课程页：存在未来课节且未绑定主邮箱时展示课程提醒卡
5. 课程详情页：下一节课临近且未绑定主邮箱时展示强提示
6. 课后总结页：学生未配置第二邮箱时展示“添加家长邮箱”引导

## 5. 邮箱提醒设置页原型

## 5.1 页面结构

```text
┌─────────────────────────────┐
│ 邮箱提醒设置                 │
│ 绑定邮箱后，平台会在关键时刻 │
│ 用邮件提醒你，不再依赖短信。 │
├─────────────────────────────┤
│ 主邮箱                       │
│ 用于：未读消息、开课提醒、课后总结 │
│ [ li***@gmail.com ] 已验证    │
│ [ 修改邮箱 ]                 │
├─────────────────────────────┤
│ 课后总结邮箱                 │
│ 适合填写家长邮箱             │
│ 仅接收每节课结束后的课程总结 │
│ 不接收聊天消息和开课提醒     │
│ [ 输入邮箱 ] [ 获取验证码 ]   │
│ [ 输入验证码 ] [ 完成设置 ]   │
└─────────────────────────────┘
```

### 5.2 顶部说明

建议文案：

`由于短信目前仅用于验证码，课程和消息提醒将主要通过邮箱发送。绑定后，你可以及时收到未读消息提醒、开课提醒和课后总结。`

### 5.3 主邮箱模块

标题：

`主邮箱`

说明：

`用于接收未读消息提醒、开课提醒和课后总结。`

未绑定状态：

1. 邮箱输入框
2. 获取验证码按钮
3. 验证码输入框
4. 完成绑定按钮

已绑定状态：

1. 脱敏邮箱
2. 已验证标签
3. 修改邮箱按钮

待验证状态：

1. 当前待验证邮箱
2. 验证码输入框
3. 重新获取验证码
4. 完成验证按钮

### 5.4 课后总结邮箱模块

该模块仅学生用户展示。

标题：

`课后总结邮箱`

辅助标签：

`选填`

核心说明：

`适合填写家长邮箱。这个邮箱只会接收每节课结束后的课后总结，不会收到聊天消息、开课提醒，也不能用于登录账号。`

视觉建议：

1. 使用浅色信息卡与主邮箱区分。
2. 加一个小标签：`仅课后总结`
3. 不使用“备用邮箱”这个词，避免误解为账号安全邮箱。
4. 删除按钮文案使用“移除课后总结邮箱”，不要写“解绑账号邮箱”。

## 6. 邮箱绑定交互状态

## 6.1 输入校验

邮箱格式错误：

`请输入正确的邮箱地址`

验证码为空：

`请输入邮箱验证码`

验证码错误：

`验证码不正确，请重新输入`

验证码过期：

`验证码已过期，请重新获取`

发送过快：

`验证码已发送，请 60 秒后再试`

### 6.2 成功提示

主邮箱绑定成功：

`主邮箱绑定成功，后续将通过邮箱接收消息、开课和课后总结提醒。`

第二邮箱设置成功：

`课后总结邮箱设置成功，后续每节课总结也会发送到这个邮箱。`

修改成功：

`邮箱修改成功。`

删除第二邮箱成功：

`已移除课后总结邮箱。`

## 7. 聊天页改动

## 7.1 聊天列表页提示

触发条件：

1. 用户主邮箱未绑定或未验证。
2. 用户存在会话或进入消息中心。
3. 频控未命中。

位置：

消息列表顶部，搜索框下方。

原型：

```text
┌─────────────────────────────┐
│ 绑定邮箱，2 小时未读消息会提醒你 │
│ 不再错过老师/家长的重要回复     │
│ [立即绑定] [稍后]              │
└─────────────────────────────┘
```

说明：

1. 点击“立即绑定”跳转 `/account/email?scene=CHAT`。
2. 点击“稍后”后 7 天内不再展示同类提示。

## 7.2 聊天详情页提示

轻提示版本：

```text
绑定邮箱，未读消息可通过邮件提醒你。立即绑定
```

展示策略：

1. 仅首次进入聊天详情页展示。
2. 若聊天列表已展示且用户关闭，详情页不重复展示。

## 8. 我的课程页改动

## 8.1 课程列表顶部提示

触发条件：

1. 用户主邮箱未绑定。
2. 用户存在未来课节。

原型：

```text
┌─────────────────────────────┐
│ 开课提醒将通过邮箱发送        │
│ 绑定主邮箱后，可按站内提醒时间同步收到邮件提醒。 │
│ [去绑定邮箱]                  │
└─────────────────────────────┘
```

视觉建议：

1. 使用课程页现有卡片风格。
2. 不做强弹窗，避免影响课程浏览。
3. 若下一节课在 24 小时内开始，可升级为更醒目的浅黄色提示。

## 8.2 课程卡片内提示

若下一节课临近且未绑定主邮箱，在课程卡片下方增加：

```text
你还未绑定邮箱，可能错过本节课的开课提醒。
```

按钮：

`绑定邮箱`

## 11. 页面路由与 API 对接

## 11.1 新增前端 API

建议在 `ai-tutor-web/src/api/user.ts` 增加：

```ts
export const userEmailApi = {
  getEmailStatus: () => request.get<UserEmailStatusVO>('/user/email'),
  sendEmailCode: (payload: SendEmailCodeReq) => request.post<EmailCodeResp>('/user/email/code', payload),
  verifyEmail: (payload: VerifyEmailReq) => request.post<UserEmailItemVO>('/user/email/verify', payload),
  deleteSummaryEmail: () => request.delete<boolean>('/user/email/summary'),
  getReminderHints: (params: EmailReminderHintReq) => request.get<EmailReminderHintVO>('/user/email/reminder-hints', { params }),
}
```

类型：

```ts
type EmailType = 'PRIMARY' | 'SUMMARY_ONLY'
type EmailVerifyStatus = 'PENDING' | 'VERIFIED' | 'INVALID'

type UserEmailItemVO = {
  emailMasked: string
  verifyStatus: EmailVerifyStatus
  bounceStatus: 'NORMAL' | 'SOFT_BOUNCE' | 'HARD_BOUNCE'
  verifiedAt?: string | null
}

type UserEmailStatusVO = {
  primaryEmail?: UserEmailItemVO | null
  summaryEmail?: UserEmailItemVO | null
  canUseSummaryEmail: boolean
  tips: {
    primaryEmailMissing: boolean
    summaryEmailMissing: boolean
  }
}
```

## 11.2 页面状态管理

建议新增 composable：

`useEmailReminderStatus`

职责：

1. 拉取邮箱状态。
2. 判断是否展示提示。
3. 处理关闭提示的本地状态。
4. 统一跳转邮箱设置页。

## 13. 视觉设计建议

整体方向：

1. 主邮箱是“账号提醒能力”，视觉更稳定、正式。
2. 第二邮箱是“给家长同步总结”，视觉更温和、说明更充分。
3. 避免把第二邮箱做成安全邮箱或备用登录邮箱。

推荐模块文案：

```text
课后总结邮箱
适合填写家长邮箱
每节课结束后，我们会把课程总结同步发送到这里。
它不会收到聊天消息、开课提醒，也不能用于登录账号。
```

推荐标签：

1. `主邮箱`
2. `仅课后总结`
3. `适合家长`
4. `已验证`
5. `待验证`

## 14. 前端验收清单

### 14.1 邮箱设置页

1. 未绑定、待验证、已绑定、异常状态展示正确。
2. 主邮箱可发送验证码并完成绑定。
3. 学生可设置第二邮箱。
4. 教师不展示第二邮箱模块。
5. 第二邮箱说明清晰表达“仅接收课后总结”。

### 14.2 场景提示

1. 聊天列表未绑定主邮箱时展示未读提醒引导。
2. 我的课程存在未来课节时展示开课提醒引导。
3. 课程详情下一节课临近时展示更强提醒。
4. 课后总结页展示添加家长邮箱引导。
5. 用户关闭提示后频控生效。

### 14.3 补发体验

1. 用户补绑主邮箱后，如有可补发总结，展示补发提示。
2. 学生设置第二邮箱后，如有可补发总结，展示补发提示。
3. 不存在可补发总结时，只提示后续生效。
