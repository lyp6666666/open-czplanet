## 为什么需要修改 (Why)

当前小程序的 UI 界面过时且视觉效果不佳，特别是登录页面和导航栏，严重影响了用户体验和品牌一致性。目标是将界面现代化，使其与现有的 Web 应用设计语言保持一致，提升可用性和美观度。

## 修改内容 (What Changes)

-   **全局样式更新**：实施新的配色方案（青色 #00bebd，灰色 #f6f7fb）和与 Web 应用一致的排版系统。
-   **登录/认证重设计**：彻底重构登录界面，使用基于卡片的布局，层次分明，视觉效果更好。
-   **导航栏**：自定义底部标签栏以符合新品牌形象（选中颜色、图标）。
-   **页面布局**：重构主要页面（首页、消息、我的），使用一致的间距、卡片和组件样式。
-   **组件库**：引入符合新设计系统的可复用 UI 组件（按钮、输入框、卡片）。

## 能力 (Capabilities)

### 新增能力 (New Capabilities)
- `ui-design-system` (UI 设计系统)：定义全局设计令牌（颜色、排版、间距）和核心 UI 组件。
- `auth-flow-ui` (认证流程 UI)：指定认证过程（登录、注册、角色切换）的用户界面和交互。
