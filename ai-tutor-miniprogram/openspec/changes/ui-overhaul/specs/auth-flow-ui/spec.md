## 新增需求 (ADDED Requirements)

### 需求：登录界面布局 (Login Interface Layout)
当用户未登录时，应用必须在“我的”标签页内或作为模态层展示专门的登录界面。
界面必须包含：
-   一个干净的卡片式容器，白色背景，圆角 (`18px`)。
-   醒目的“登录/注册”标题。
-   圆角边框 (`12px`) 且间距得当的手机号和验证码输入框。
-   全宽且背景为主色 (`#00bebd`) 的“登录/注册”主操作按钮。
-   “微信登录”的次要操作按钮。

#### 场景：未认证用户视图 (Unauthenticated User View)
-   **当** 用户导航到“我的”标签页且未登录时
-   **则** 系统必须显示重新设计的登录卡片
-   **且** 卡片外的背景（如果有）必须是主色或与 Web 应用风格匹配的渐变色

#### 场景：输入框聚焦状态 (Input Focus State)
-   **当** 用户聚焦于输入框（手机号或验证码）时
-   **则** 输入框边框颜色必须变为主色 (`#00bebd`)

### 需求：认证反馈 (Authentication Feedback)
应用必须在认证过程中提供清晰的视觉反馈。
-   “发送验证码”按钮在激活时必须显示倒计时。
-   网络请求期间，按钮必须显示加载状态。
-   错误信息必须清晰显示（例如，Toast 提示或行内错误）。

#### 场景：发送验证码 (Sending Verification Code)
-   **当** 用户点击“发送验证码”时
-   **则** 按钮必须被禁用并显示 60 秒倒计时
-   **且** 成功时必须显示 Toast 消息“验证码已发送”

#### 场景：登录操作 (Login Action)
-   **当** 用户点击“登录”按钮时
-   **则** 按钮必须显示加载指示器
-   **且** 成功后，视图必须过渡到已认证的“我的”个人资料视图
