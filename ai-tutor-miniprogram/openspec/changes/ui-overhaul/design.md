## 背景 (Context)

当前小程序使用 `uView Plus` 的默认样式，导致外观平淡且缺乏品牌特色。登录体验虽然功能正常，但缺乏视觉吸引力。Web 应用 (`ai-tutor-web`) 具有独特的设计系统（青色 #00bebd，卡片式布局），需要将其移植到小程序以保持品牌一致性。

## 目标 / 非目标 (Goals / Non-Goals)

**目标：**
-   将小程序的视觉识别与 Web 应用统一。
-   为登录/注册流程实施现代化的卡片式 UI。
-   使用全局 CSS 变量标准化颜色、排版和间距。
-   重构底部导航栏以匹配新的配色方案。

**非目标：**
-   重构底层业务逻辑或 API 交互（除非依赖于 UI）。
-   完全替换 `uView Plus`（我们将在其基础上进行样式覆盖）。

## 决策 (Decisions)

### 1. 全局样式策略
-   **决策**：将 Web 应用的 `global.css` 中的 CSS 变量移植到 `uni.scss`。
-   **理由**：确保设计令牌的单一来源，简化维护。
-   **实施**：在 `uni.scss` 中添加 `--primary`, `--text`, `--bg` 等变量，并在 `App.vue` 中应用。

### 2. 登录 UI 实现
-   **决策**：重构 `pages/me/index.vue` 作为认证入口点。
-   **理由**：当前实现根据登录状态切换内容。我们将增强“未登录”状态，使其看起来像一个专门的高质量登录屏幕（参考 Web 应用的 `AuthPage.vue`）。
-   **细节**：在登录状态下使用带有青色背景的全屏容器，登录后过渡到标准布局。

### 3. 导航栏
-   **决策**：使用新配色方案配置 `pages.json` 的 `tabBar`。
-   **理由**：原生标签栏比自定义组件标签栏提供更好的性能和兼容性。

### 4. 组件样式
-   **决策**：尽可能全局覆盖 `uView Plus` 样式，并对特定页面使用 scoped SCSS 覆盖。
-   **理由**：`uView Plus` 提供了良好的功能，但需要视觉调整。

## 风险 / 权衡 (Risks / Trade-offs)

-   **风险**：`uView Plus` 组件样式可能难以完全覆盖。
    -   **缓解措施**：使用深度选择器 (`::v-deep`) 和自定义工具类。如果组件太僵化，则用自定义 Vue 组件替换（例如，简单的卡片或按钮）。
-   **风险**：小程序中的导航栏定制受限。
    -   **缓解措施**：充分利用 `pages.json` 中的可用配置选项（颜色、图标）。如果绝对必要，实施自定义标签栏，但首选原生以保证稳定性。
